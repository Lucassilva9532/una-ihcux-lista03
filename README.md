# una-ihcux-lista03
 Visibilidade do estado do sistema

Essa heurística afirma que o sistema deve sempre manter o usuário informado sobre o que está acontecendo, por meio de feedback adequado e em tempo razoável.

 Objetivo

Garantir que o usuário saiba:

Onde está no sistema
O que está acontecendo no momento
Qual foi o resultado de suas ações
 Princípio

Interfaces devem comunicar claramente seu estado atual, evitando incerteza ou confusão.

 Boas práticas
 
Exibir mensagens de feedback imediato após ações (ex: “Salvo com sucesso”)
Mostrar indicadores de carregamento (spinners, barras de progresso)
Destacar elementos ativos ou selecionados
Atualizar a interface em tempo real quando possível
 Problemas comuns
 
 
Falta de resposta após uma ação do usuário
Processos longos sem indicação de progresso
Mudanças de estado sem aviso
Exemplos

Barra de progresso ao enviar um arquivo
Indicador “digitando…” em chats
Alteração visual de botão ao ser clicado
 Benefícios
 
Reduz ansiedade e incerteza
Aumenta a confiança no sistema
Melhora a experiência do usuário (UX)
